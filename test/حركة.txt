تمام، خليني أفصلها معك بشكل عملي أكتر — أنواع الحركات، وكيف تتحكم فيها بالتفصيل.

## 1. أشهر أنواع الـ Animations المتوفرة بالـ package

| Class | التأثير |
|---|---|
| `animate-fade-in` | ظهور تدريجي بالمكان (بدون حركة مكانية) |
| `animate-fade-out` | اختفاء تدريجي |
| `animate-slide-in-bottom` | يدخل من تحت لفوق |
| `animate-slide-in-top` | يدخل من فوق لتحت |
| `animate-slide-in-left` | يدخل من اليسار |
| `animate-slide-in-right` | يدخل من اليمين |
| `animate-zoom-in` | يكبر من الصغر للحجم الطبيعي |
| `animate-zoom-out` | يصغر من الحجم الطبيعي |
| `animate-blurred-fade-in` | ظهور تدريجي مع تمويه (blur) يزول تدريجيًا |
| `animate-bounce-in` | يدخل بحركة "نطة" خفيفة |

## 2. التحكم بـ Delay (متى تبلش الحركة)

عندك قيم جاهزة:
```
animate-delay-0
animate-delay-100
animate-delay-150
animate-delay-200
animate-delay-250
animate-delay-300
animate-delay-400
animate-delay-500
animate-delay-700
animate-delay-800
animate-delay-900
animate-delay-1000
```

كل رقم = ملي ثانية. يعني `animate-delay-300` = العنصر ينتظر 300ms قبل ما يبلش يتحرك.

### قاعدة عملية للـ Stagger (التتابع):
لو عندك N عناصر، خلي الفرق بين كل عنصر والتاني ثابت (مثلاً 100-150ms):
```
عنصر 1 → delay-0
عنصر 2 → delay-100
عنصر 3 → delay-200
عنصر 4 → delay-300
```
كل ما زاد عدد العناصر أكتر، خلي الفرق أصغر (50-100ms) عشان القائمة كلها ما تاخد وقت طويل تظهر بالكامل.

## 3. التحكم بـ Duration (سرعة الحركة نفسها)

```
animate-duration-faster
animate-duration-fast
animate-duration-normal
animate-duration-slow
animate-duration-slower
```

أو بالأرقام المباشرة (زي الـ delay): `animate-duration-300`, `animate-duration-500`, إلخ.

## 4. مثال متكامل — قائمة بطاقات تظهر بالتتابع مع حركة مختلفة عن الأولى

```html
<div class="grid grid-cols-3 gap-6">

  <div class="animate-slide-in-bottom animate-delay-0 animate-duration-normal">
    <div class="bg-white rounded-xl p-6 shadow">بطاقة 1</div>
  </div>

  <div class="animate-slide-in-bottom animate-delay-150 animate-duration-normal">
    <div class="bg-white rounded-xl p-6 shadow">بطاقة 2</div>
  </div>

  <div class="animate-slide-in-bottom animate-delay-300 animate-duration-normal">
    <div class="bg-white rounded-xl p-6 shadow">بطاقة 3</div>
  </div>

</div>
```

## 5. لو بدك الحركة تصير بس لما العنصر يظهر بالشاشة أثناء الـ Scroll (مو أول ما تفتح الصفحة)

هون بتستخدم `view-animate-single` بدل `animate-*` العادي:

```html
<div class="view-animate-single fade-in">
  هاد العنصر رح يتحرك بس لما توصله بالـ scroll
</div>
```

هيك الحركة رح تصير تلقائيًا كل ما العنصر يدخل ضمن الشاشة المرئية (viewport)، مفيد جدًا لصفحات Landing الطويلة.

## 6. خلاصة الفرق بين الـ 3 محاور يلي بتتحكم فيهم:

| المحور | الكلاس | وظيفته |
|---|---|---|
| نوع الحركة | `animate-fade-in`, `animate-slide-in-*`... | شكل الحركة نفسها |
| التوقيت | `animate-delay-*` | متى تبلش |
| السرعة | `animate-duration-*` | كم بتاخد وقت |

جرب دمج الأنواع المختلفة مع بعض (مثلاً `zoom-in` للعنصر الأول و`slide-in-left` للتاني) وشوف الفرق، وقلي لو حبيت تركيبة معينة وما عرفت تظبطها.
